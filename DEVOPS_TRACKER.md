## 1. Infra status

### 1.1 Outage discovered 2026-07-10

`alecrae.com`, `mail.alecrae.com`, and `status.alecrae.com` were all found
returning **HTTP 503** simultaneously during this session (confirmed via
direct fetch, not just a report). All three route through the same
`vapron-bun-gateway` on one box to two local ports (4100 api, 4200 web) per
`docs/infra/box-deploy.md` — a single-box, single-point-of-failure
architecture where one gateway/box problem takes every subdomain down at
once. Root cause not yet confirmed (blocked on box access — see below).
**No postmortem written yet** — per CLAUDE.md's Emergency Protocol, one
should land in `docs/postmortems/` once root cause is confirmed.

### 1.2 Box migration (149.28.119.158 → 66.42.121.161 "Jarvis")

Per the user, **Jarvis (66.42.121.161) is now the box that actually controls
alecrae.com** — `docs/infra/box-deploy.md` (last updated 2026-06-15) is
stale and still describes 149.28.119.158 as production; it needs a rewrite
once box access and the outage are sorted.

### 1.3 Box access (in progress, 2026-07-10)

No previously-working SSH credential was on hand for Jarvis this session.
Tailscale was installed on the operator's machine and joined to the
existing tailnet (both `jarvis` and `vapron-158` are visible peers), but as
of this update Jarvis does not yet advertise Tailscale SSH capability —
`sudo tailscale set --ssh` needs to be run **on Jarvis itself** (console/
provider access, not the client machine) before Tailscale SSH can be used
in place of key-based auth. Once working, save the confirmed method
(alias, key, or Tailscale hostname) as a persistent memory entry so future
sessions don't have to rediscover it.

### 1.7 Reliability gap: no staging environment

Tonight's outage exposed that there is **no staging environment anywhere in
this stack** — production is a single box with a manual `git pull` ritual,
so every merge to `main` is one deploy away from being live with nothing in
between to catch breakage. The box in question reportedly serves multiple
products beyond alecrae.com, which raises the stakes further. **Recommend a
dedicated, separate conversation to design a proper staging tier** (likely
spanning infrastructure beyond this repo) rather than architecting it
inline here — flagged, not actioned.

## 3. Prioritized gap-closing roadmap (for whenever alecrae.com resumes)

Grouped by domain, not a flat list of 61 routes. Effort estimates are rough
and directional.

1. **Security Intelligence** (`security-intelligence.ts`, ~12 endpoints,
   partially wired) — HIGH impact: trust-critical, naturally visible as a
   dashboard/policy page. ~3-4 days to finish wiring.
2. **Contact Enrichment** (`contact-enrichment.ts`) — HIGH impact, LOW
   effort: the contacts page already exists and is wired; enrichment is an
   incremental add-on. ~1-2 days.
3. **Delegation / Collaboration** (`delegation.ts`, `collaborate.ts`) —
   MEDIUM-HIGH impact for team/business-tier users, ties into the
   multi-workspace work already shipped (Known Issue #63). ~4-5 days.
4. **Workflow Automation** (`workflows.ts` — already wired; remaining gap is
   deeper automation domains) — MEDIUM impact, power-user feature.
5. **Analytics subsystems** (`analytics-dashboard.ts`,
   `scheduling-analytics.ts`, `sentiment-timeline.ts`,
   `productivity-analytics.ts`) — MEDIUM impact, lower urgency since core
   analytics/heatmap are already wired. ~1-2 days per subsystem.
6. **AI services stack** (`ai-intelligence.ts`, `ai-writing.ts`,
   `context-intelligence.ts`, `search-intelligence.ts`,
   `knowledge-graph.ts`) — MEDIUM impact individually, HIGH cumulative
   differentiation ("AI-native" is the core pitch per CLAUDE.md's Mission).
   Largest remaining domain — knowledge-graph alone likely needs a graph
   visualization component. Recommend its own multi-week workstream.
7. **Voice features** (`voice.ts`, `dictation.ts`, `meeting-link.ts`,
   `video-meetings.ts`) — LOWER priority; base voice page already exists,
   remaining gap is the advanced tier.
